# 5 Minute Tutorial

This tutorial walks you through creating a simple React app with Remirror's ready-to-use WYSIWYG Editor. You'll have a fully functional rich text editor running in just a few minutes.

## Prerequisites

You need to have a recent version of Node.js installed (v14.21.3 or higher).

## Step-by-Step Guide

<Steps>
  <Step title="Create a React app">
    First, let's set up a basic React app with [Vite](https://vitejs.dev/guide/):

    ```bash
    npx create-vite vite-remirror --template react-ts
    cd vite-remirror
    ```

    This creates a new React + TypeScript project using Vite as the build tool.
  </Step>

  <Step title="Install Remirror packages">
    Install Remirror and its peer dependencies:

    <CodeGroup>
    ```bash npm
    npm add remirror @remirror/react @remirror/pm @emotion/react @emotion/styled @remirror/react-editors
    ```

    ```bash yarn
    yarn add remirror @remirror/react @remirror/pm @emotion/react @emotion/styled @remirror/react-editors
    ```

    ```bash pnpm
    pnpm add remirror @remirror/react @remirror/pm @emotion/react @emotion/styled @remirror/react-editors
    ```
    </CodeGroup>

    <Note>
      The `@emotion/react` and `@emotion/styled` packages are required for the pre-built editor components. If you're building a custom editor from scratch, you may not need them.
    </Note>
  </Step>

  <Step title="Create your first editor">
    Replace the contents of `src/App.tsx` with the following code:

    ```tsx src/App.tsx
    import { WysiwygEditor } from '@remirror/react-editors/wysiwyg';

    const App: React.FC = () => {
      return (
        <div style={{ padding: 16 }}>
          <WysiwygEditor placeholder='Enter text...' />
        </div>
      );
    };

    export default App;
    ```

    That's it! The `WysiwygEditor` component comes with everything you need: formatting buttons, keyboard shortcuts, and markdown input rules.
  </Step>

  <Step title="Start the development server">
    Run the app:

    ```bash
    npm run dev
    ```

    <Tip>
      Open your browser and navigate to the URL shown in the terminal (usually `http://localhost:5173`). You should see a fully functional WYSIWYG editor!
    </Tip>
  </Step>
</Steps>

## Working with Editor Data

Now that you have a working editor, let's learn how to get data out and put data in.

<Tip>
  Using **JSON** is recommended for storing editor content. It's simpler for data migrations, search indexing, and other programmatic operations compared to HTML.
</Tip>

### Capturing Content Changes

Remirror provides components that let you react to content changes:

- **`OnChangeJSON`** (Recommended): Provides a JSON representation of the document
- **`OnChangeHTML`**: Provides an HTML string representation

```tsx src/App.tsx
import { useCallback } from 'react';
import type { RemirrorJSON } from 'remirror';
import { OnChangeJSON } from '@remirror/react';
import { WysiwygEditor } from '@remirror/react-editors/wysiwyg';

const STORAGE_KEY = 'remirror-editor-content';

const App: React.FC = () => {
  const handleEditorChange = useCallback((json: RemirrorJSON) => {
    // Store the JSON in localStorage
    window.localStorage.setItem(STORAGE_KEY, JSON.stringify(json));
  }, []);

  return <MyEditor onChange={handleEditorChange} />;
};

interface MyEditorProps {
  onChange: (json: RemirrorJSON) => void;
}

const MyEditor: React.FC<MyEditorProps> = ({ onChange }) => {
  return (
    <div style={{ padding: 16 }}>
      <WysiwygEditor placeholder='Enter text...'>
        <OnChangeJSON onChange={onChange} />
      </WysiwygEditor>
    </div>
  );
};

export default App;
```

<Note>
  In this example, we're using `localStorage` for simplicity. In a real application, you'd typically save to a database via an API call.
</Note>

### Setting Initial Content

To load saved content back into the editor, use the `initialContent` prop:

```tsx src/App.tsx
import { useCallback, useState } from 'react';
import type { RemirrorJSON } from 'remirror';
import { OnChangeJSON } from '@remirror/react';
import { WysiwygEditor } from '@remirror/react-editors/wysiwyg';

const STORAGE_KEY = 'remirror-editor-content';

const App: React.FC = () => {
  const [initialContent] = useState<RemirrorJSON | undefined>(() => {
    // Retrieve the JSON from localStorage (or undefined if not found)
    const content = window.localStorage.getItem(STORAGE_KEY);
    return content ? JSON.parse(content) : undefined;
  });

  const handleEditorChange = useCallback((json: RemirrorJSON) => {
    // Store the JSON in localStorage
    window.localStorage.setItem(STORAGE_KEY, JSON.stringify(json));
  }, []);

  return <MyEditor onChange={handleEditorChange} initialContent={initialContent} />;
};

interface MyEditorProps {
  onChange: (json: RemirrorJSON) => void;
  initialContent?: RemirrorJSON;
}

const MyEditor: React.FC<MyEditorProps> = ({ onChange, initialContent }) => {
  return (
    <div style={{ padding: 16 }}>
      <WysiwygEditor placeholder='Enter text...' initialContent={initialContent}>
        <OnChangeJSON onChange={onChange} />
      </WysiwygEditor>
    </div>
  );
};

export default App;
```

## Understanding the JSON Format

The `RemirrorJSON` type represents the ProseMirror document structure. Here's what it looks like:

```json
{
  "type": "doc",
  "content": [
    {
      "type": "paragraph",
      "content": [
        { "type": "text", "text": "Hello " },
        {
          "type": "text",
          "text": "world",
          "marks": [{ "type": "bold" }]
        }
      ]
    }
  ]
}
```

This structure preserves all formatting, structure, and metadata in your document.
